About Reverse DNS Lookup

Reverse DNS (rDNS) is the process of resolving an IP address back to a hostname using a PTR record. While a forward DNS lookup maps a name to an IP, a reverse lookup does the opposite — asking "what hostname is registered for this IP?". This is useful for identifying server ownership, verifying email sender identity (many spam filters check PTR records), and diagnosing network issues. Both IPv4 and IPv6 are supported: enter any address in standard notation (e.g. 8.8.8.8 or 2606:4700:4700::1111) and the tool queries the appropriate in-addr.arpa or ip6.arpa zone. Not all IPs have a PTR record — many cloud and hosting providers leave them unset.
